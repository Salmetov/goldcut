# goldcut

Выуживает «золото» из длинных видео на YouTube и готовит из него короткие вертикальные клипы для TikTok.

Ты кидаешь ссылку Telegram-боту → система анализирует транскрипт, **отбирает самые ценные самодостаточные куски** (а не режет ролик подряд) → предлагает их с таймкодами и саммари → ты выбираешь → получаешь готовые клипы 9:16 с субтитрами + текст и хэштеги → публикуешь черновиком в TikTok.

## Главный принцип

«Нарезка» и «отбор золота» — **две разные задачи**:
1. **Сегментация** — где границы законченных мыслей (по смыслу, не по таймеру).
2. **Скоринг** — какие из них реально стоят клипа (берём top-K, большая часть ролика не даёт ничего).

См. [ARCHITECTURE.md](ARCHITECTURE.md) — подробно про компоненты и поток данных.

## Компоненты (system blocks)

| Блок | Где живёт | Отвечает за |
|------|-----------|-------------|
| **fetcher** | Mac (residential IP) через Tailscale | `yt-dlp`: субтитры, heatmap, скачивание выбранных отрезков |
| **analyzer** | сервер | LLM-сегментация транскрипта + скоринг «золотистости» |
| **cutter** | Mac / сервер | `ffmpeg`: вырез, 9:16, вшивание субтитров |
| **bot** | сервер | Telegram-оркестратор: диалог, выбор клипов, состояние |
| **publisher** | сервер | подготовка черновика для TikTok |

Почему добыча именно на Mac через Tailscale — см. [docs/decisions/0001-fetch-via-mac-over-tailscale.md](docs/decisions/0001-fetch-via-mac-over-tailscale.md).
