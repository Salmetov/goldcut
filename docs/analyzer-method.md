# Метод отбора «золота» (analyzer)

Спецификация того, как goldcut выбирает короткие клипы из длинного ролика.
Реализация — `src/goldcut/analyzer/segmenter.py`; этот документ — её источник правды.

## Принцип: две разные задачи

«Нарезка» и «отбор золота» — НЕ одно и то же. Наивный путь (порезать транскрипт на
равные чанки и засаммаризировать) даёт последовательную кашу. Мы делаем два слоя:

1. **Сегментация (LLM).** Найти отдельные самодостаточные мысли — границы по смыслу и
   предложениям, а не по таймеру.
2. **Скоринг (код).** Оценить каждую по рубрике, добавить heatmap-буст, взять top-K.
   Большая часть ролика обязана не дать ничего — это и есть «выуживание».

Разделение намеренное: веса и heatmap-математика живут в коде (прозрачно и тюнится),
LLM отвечает только за нахождение кусков и оценки по рубрике.

## Ключевой принцип: LLM выбирает, а не генерирует

Первая версия давала LLM прозу с метками `[MM:SS]` и просила вернуть таймкоды и текст
клипа. Это ломалось системно: LLM оценивал секунды на глаз, пересказывал текст
(не дословно) — и никакая привязка/починка границ не давала целых мыслей.

Текущая схема — **выбор диапазонов предложений**:

1. Код строит из пословных таймкодов VTT **пронумерованные предложения**
   (`transcript.build_sentences`): границы по пунктуации авто-сабов, паузам в речи
   (>1.2с) и лимиту длины. У каждого предложения — точные start/end секунды.
2. LLM получает список `S{i} [MM:SS] текст` (обрывки без финальной пунктуации помечены `⋯`)
   и возвращает клип как **диапазон**: `first_sentence`/`last_sentence` + `title`,
   `summary`, `hook`, `scores`, `why` (structured output, `SegmentSelection`).
3. Код берёт таймкоды и дословный текст из диапазона. Дрейфовать нечему по построению:
   ни снапинга, ни выравнивания текста в основном пути нет.
4. Страховки в коде: диапазон с хвостом-обрывком (без финальной пунктуации)
   дотягивается до конца фразы (≤2 предложений / ≤6с); клипы вне 10–90с отбрасываются.

## Editorial-бриф (требования к диапазону)

- **15–60 секунд**; **мысль целиком** — если ключевая фраза опирается на контекст
  («this», «that experiment», ответ на вопрос ведущего), сетап включается в диапазон;
  проверка глазами зрителя, не смотревшего ролик;
- первое предложение — **хук**, последнее — **развязка**; не начинать с вялых подводок;
- **исключается**: реклама/спонсорские чтения, само-промо, вступительный трейлер-нарезка,
  организационная болтовня, повторы;
- **разнообразие** — разные мысли из разных мест ролика;
- heatmap — **сигнал, а не диктат**.

## Рубрика и веса (скоринг в коде)

Каждый кандидат получает оценки 0–5 от LLM; код считает взвешенный балл:

| Критерий | Что оценивает | Вес |
|----------|---------------|-----|
| `self_contained` | понятно без контекста ролика | 0.25 |
| `insight` | неочевидность мысли | 0.25 |
| `hook` | сила первых ~3 секунд | 0.20 |
| `emotion` | эмоция / цитируемость | 0.15 |
| `payoff` | есть развязка, не обрывается | 0.15 |

**heatmap-буст:** макс. интенсивность heatmap (0..1), попадающая в `[start_s, end_s]`,
умноженная на `HEATMAP_SCALE` (0.5), прибавляется к взвешенному баллу.

`total = Σ(вес × оценка) + heatmap_boost`. Сортировка по `total` ↓, отдаём top-K.

Веса, `HEATMAP_SCALE`, `bucket_s` парсера и сам бриф — точки тюнинга по результатам прогонов.

## История подходов (почему так)

- **v1: LLM возвращает таймкоды** — секунды дрейфовали до минут (оценка «на глаз»
  по 20с-абзацам). Клип «как работает ИИ» уезжал на 65:36 вместо 40:10.
- **v2: привязка текста клипа к пословному VTT** (`snap_timecodes`, difflib с якорем
  на длиннейшем блоке) + расширение до границ предложений. Починило секунды, но не
  смысл: LLM пересказывал текст (не дословно), абзацная разбивка резала предложения,
  а «граница предложения» ≠ «граница мысли» (клип «consent» терял сетап про напиток).
- **v3 (текущая): выбор диапазонов предложений** — см. выше. Утилиты v2
  (`snap_timecodes`, `extend_to_sentence_bounds`) остаются в `transcript.py` для
  будущих ручных правок границ.

## Модель

`claude-opus-4-8` по умолчанию (тонкость отбора важнее цены; ~20k токенов транскрипта
влезают одним запросом). Настраивается через `GOLDCUT_ANALYZER_MODEL`. Вызов —
официальный Anthropic SDK, `client.messages.parse(... output_format=SegmentSelection)`
(валидированный structured output, без ручного парсинга JSON).

## Валидация подхода (2026-06-25)

Прогон вручную на 88-мин подкасте (Diary Of A CEO × Roman Yampolskiy): heatmap-пики
совпали с независимым отбором (главный пик ролика 53:27 = клип №1), реклама корректно
отсеяна, из 88 минут выбрано ~11 кусков по 30–60с в разных местах. Подтвердило, что
связка «LLM-редактор + heatmap-буст» находит золото, а не угадывает.
