# ADR 0001 — Добыча видео через Mac по Tailscale

**Статус:** принято · **Дата:** 2026-06-25

## Контекст

YouTube блокирует запросы с IP дата-центров («confirm you're not a bot», `RequestBlocked`).
Это подтверждено эмпирически: и временная среда, и прод-сервер OKO (IP дата-центра) получают блок.
Значит, добыча с не-резидентного IP — постоянная проблема, а не разовая.

## Решение

Шаг добычи (`yt-dlp`) выполняется на **Mac пользователя** (`macbook-air-muzaffar`), у которого
резидентный IP. Сервер общается с Mac по приватной сети **Tailscale** (tailnet `tail9ae71e.ts.net`),
наружу ничего не торчит.

Добыча изолирована за интерфейсом `Fetcher`, поэтому бэкенд можно сменить (residential-прокси,
телефон, cookies) без правок остального кода.

## Топология

- **Mac** — `fetcher.worker` (HTTP на Tailscale-интерфейсе) + `yt-dlp`/`ffmpeg`.
- **Сервер** (оркестратор/бот) — `fetcher.client` дёргает воркер по tailnet.
- Связь — только внутри tailnet.

## Настройка Tailscale (штатный шаг, не интерактивно)

На сервере headless-логин через URL ненадёжен. Правильный путь — **tagged ephemeral auth-key**:

```bash
curl -fsSL https://tailscale.com/install.sh | sh
tailscale up --authkey=tskey-... --hostname=goldcut-oko --ssh
```

На Mac — включить приём по SSH:

```bash
tailscale up --ssh
```

## Последствия

- ✅ Блокировка по IP снимается в принципе — сервер не ходит в YouTube.
- ✅ Резидентный IP пользователя, без месячной платы за прокси.
- ⚠️ Mac — ноутбук: нужен аптайм (`caffeinate`, не давать спать) для роли постоянного узла.
- ⚠️ Лимит Telegram Bot API 20 МБ → для крупных файлов поднять self-hosted Bot API сервер (до 2 ГБ),
  либо качать только короткие отрезки.

- Mac уже в tailnet (`100.119.65.77`).
