# ADR 0002 — Полная закачка на Mac, нарезка на сервере

**Статус:** принято · **Дата:** 2026-07-02

## Контекст

Для нарезки отрезков `yt-dlp --download-sections` требует ffmpeg **на Mac** — а там нет
pip/brew, и установка стороннего бинарника требует ручного шага пользователя. Кроме того,
без JS-рантайма (deno) YouTube отдаёт только прогрессивный формат 18 (360p mp4) —
остальные форматы упираются в n-challenge / PoToken / SABR / DRM-эксперименты.

## Решение

Основной поток видео — **полная закачка + серверная нарезка**:

1. `POST /download {url}` — воркер на Mac скачивает полный mp4 (формат `b[ext=mp4]/b`)
   в кэш `~/goldcut-worker/cache/<id>.mp4`. Идемпотентно.
2. `GET /video/{id}` — сервер забирает файл по tailnet в свой кэш (`/root/goldcut/cache`).
3. Все нарезки (9:16, сабы, любые правки границ) — **локально на сервере** через ffmpeg (apt).

Воркер на Mac — **чистый stdlib** (`http.server` + standalone-бинарь yt-dlp), под launchd
(`~/Library/LaunchAgents/com.goldcut.fetcher.plist`, RunAtLoad+KeepAlive).

## Почему так

- ✅ Работает без ffmpeg на Mac и без ручных шагов.
- ✅ Одна закачка с YouTube на видео → сколько угодно нарезок/перенарезок локально,
  без повторных походов в YouTube (меньше следов, быстрее итерации).
- ✅ Нарезка из локального файла точнее (`-ss/-to` по кэшированному mp4), чем
  `--download-sections` по сети.
- ⚠️ Разовый перенос ~230MB на 1.5ч ролика по tailnet (~105с при прямом соединении) — приемлемо.

## Ограничения и апгрейды

- **Качество: 360p (формат 18).** Для выше нужен JS-рантайм на Mac (deno/node) —
  тогда yt-dlp решает n-challenge и открываются 720p+. Установка: `brew install deno`
  или https://deno.land — ручной шаг пользователя. Воркер подхватит автоматически.
- `POST /cut` (нарезка на Mac) оставлен в воркере как опциональный путь — активируется,
  если положить ffmpeg в `~/bin/ffmpeg` на Mac.
